# node.js
- 구글 크롬 브라우저에서 JS 코드를 처리하기 위한 v8엔진을 사용하는 서버 프로그램 개발을 위한 플랫폼
- ^^빠르고 확장 가능^^한 네트워크 애플리케이션을 쉽게 구축
	- 단일 스레드지만 확장성이 뛰어남. 톰캣보다 확장성이 좋고 더 많은 클라이언트 요청 처리 가능
	- 여러 스레드: 하나의 프로세스에서 여러 작업을 왔다갔다하며 처리
	- 단일 스레드: 하나의 프로세스가 하나의 작업을 담당. 여러 작업 하려면 여러 프로세스 필요
- 가볍고 효율적인 ^^비동기식^^ 이벤트 처리
- 한번 작성한 프로그램은 Windows, Linux 등 다양한 플랫폼에서 동작 가능
- 버퍼링이 없음
	- 버퍼링: cpu와 입출력장치 간의 속도 차이가 있음. 그래서 메모리에 쌓아뒀다가 입출력 장치에 한번에 보냄. 속도가 느려짐.
	- node.js는 버퍼링 없이 그냥 바로 보냄. 빠름.
- 여러 도구들이 node.js 엔진에 올라가서 코드를 처리


# set up
- `npm init -y` 해서 package.json 만들기
- `node 파일명` 으로 바로 시작


# 모듈
- 자주 사용하는 함수를 별도의 js 파일에 만든 것
- js파일 = 하나의 모듈 = 하나의 객체 단위
-
  ```js
  exports.f2 = function() {
    console.log("f2 함수 호출");
  }
  ```
- node.js는 JS문법과 모듈 사용법만 알면 모두 학습한 거다
